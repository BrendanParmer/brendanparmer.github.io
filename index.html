Synthetic Data Generation with CARLA


Technology

  • Unreal Engine 4
  • Python

About

Today's artificial intelligence systems require a lot of data to train on. As Brunswick develops its autonomous boating capabilities, we find ourselves in need of more and more training data. This data is often hard to collect, as many of the situations we want to test for may be too dangerous for human drivers, or too rare to test too often. I explored CARLA as a potential solution to this problem. CARLA is an open-source plugin for Unreal, originally intended for virtually training autonomous cars and other land vehicles. I adapted the system for a marine environment. With this modified version, we could control boat traffic, the weather, time of day, the environment, and even more, while recording simulated sensor data like RGB cameras, stereocameras, and LiDAR sensors.

Blender add-on development


Technology

  • Python
  • Blender

About

3D computer graphics is a big passion of mine, and Blender's API makes it possible for me to combine my programming knowledge with the art world. I've released a couple add-ons so far. Block On is a procedural voxelization addon I designed to address some of the issues with Blender's in-built voxelization tool. Node To Python is an addon that allows users to convert large node groups to simple Python scripts, which makes addon development much easier when working with these node groups.

Block On

I created this add-on as a way to instantly turn any 3D object into a blocky version of itself within Blender.

Blender's built-in voxelization modifier doesn't preserve material information, and it can be somewhat slow. Block On uses Geometry Nodes to not only quickly voxelize the object, but also give the user control over more variables, including:

  • Size
  • Resolution
  • Levels Visible
  • and more!

Using Geometry Nodes allows the user to easily build on top of the node network for their own needs. The variables are fully animatable, which can produce some really neat effects.

The project is currently in version 1.1 and is compatible with Blender 3.0-3.3, and I'm actively improving it.

Node To Python

I love making things with Geometry Nodes, but there wasn't a great way of distributing these to people. However, Blender lets people code add-ons with Python scripts, which are much easier to distribute than binary .blend files. I wrote an add-on to write add-ons given a Geometry Node group.

While it only works with Geometry Nodes right now, I plan on expanding it to Blender's Shader and Compositing nodes, as well. The developers are planning Everything Nodes, in which more and more of the engine get the power of a procedural workflow, which my add-on will be helpful with.
