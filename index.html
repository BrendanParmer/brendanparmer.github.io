In January, we exhibited an immersive, photorealistic boat simulator at the Consumer Electronics Show in Las Vegas. Aside from programming gameplay and UI elements, I also had a chance to lead our procedural asset creation and sound design for the project. After CES, we'll use the Future Helm for prototyping new boat concepts, user-testing new UI features, and training new boaters in simulations.

Procedural Asset Creation

As we were designing a city for the project, we realized a lot of conventional building assets wouldn't fit into our vision of the environment. I came up with a solution to get us the style we wanted, without sacrificing too many hours to asset creation.

I chose Blender for the project because of its Geometry Nodes feature, a powerful node-based procedural modelling tool. All a user needs to do is draw a curve into the scene, and the add-on will take care of the rest. Users can tweak parameters such as the number of windows, materials, balconies, plants, height, lit windows, the building shape, and much more without having to dive into the node network. A city with hundreds of unique buildings that may've taken weeks to model beforehand now can be built within an afternoon.

Sound Design

I also have been working with Unreal Engine's MetaSounds. At the lab, we've got a 5.1 surround sound system set up, and it was fun handling the sound spatialization for our setup. I was able to control variables like pitch and volume directly from blueprints, which interfaced well with the rest of our project and made it easy to add and edit new sounds.

Block On


Technology

  • Python
  • Blender

About

I created this add-on as a way to instantly turn any 3D object into a blocky version of itself within Blender.

Blender's built-in voxelization modifier doesn't preserve material information, and it can be somewhat slow. Block On uses Geometry Nodes to not only quickly voxelize the object, but also give the user control over more variables, including:

  • Size
  • Resolution
  • Levels Visible
  • and more!

Using Geometry Nodes allows the user to easily build on top of the node network for their own needs. The variables are fully animatable, which can produce some really neat effects.

The project is currently in version 1.1 and is compatible with Blender 3.0-3.2, and I'm actively improving it. You can play around with it on my GitHub.

NodeToPython


Technology

  • Blender
  • Python

About

I love making things with Geometry Nodes, but there wasn't a great way of distributing these to people. However, Blender lets people code add-ons with Python scripts, which are much easier to distribute than binary .blend files. I wrote an add-on to write add-ons given a Geometry Node group.

While it only works with Geometry Nodes right now, I plan on expanding it to Blender's Shader and Compositing nodes, as well. The developers are planning Everything Nodes, in which more and more of the engine get the power of a procedural workflow, which my add-on will be helpful with.

You can play around with it on my GitHub
