Block On


Technology

  • Python
  • Blender

About

I created this add-on as a way to instantly turn any 3D object into a blocky version of itself within Blender.

Blender's built-in voxelization modifier doesn't preserve material information, and it can be somewhat slow. Block On uses Geometry Nodes to not only quickly voxelize the object, but also give the user control over more variables, including:

  • Size
  • Resolution
  • Levels Visible
  • and more!

Using Geometry Nodes allows the user to easily build on top of the node network for their own needs. The variables are fully animatable, which can produce some really neat effects.

The project is currently in version 1.1 and is compatible with Blender 3.0-3.2, and I'm actively improving it. You can play around with it on my GitHub.

About

What would a Discord server be without a custom bot?

Using the Discord Python API I wrote a bot to "enhance" a friend group's server. Among other things, it

  • Makes bad Dad jokes
  • Promotes our band on Spotify
  • Manages our Wordle league and score threads
  • Uses a weather API and sends a daily forecast
  • Periodically reminds members to be productive members of society
  • Responds and reacts to dozens of custom phrases

This was an early project of mine, and it's where I really started to learn how to interact with other people's software in my own code. I also learned how to host a bot on a server, as well as asynchronous programming. I still maintain it and like to add new features.

A public version is available on my GitHub.

CS 124 Course Assistant


Technology

  • Java
  • Kotlin
  • Android Studio

About

After my freshman year I spent a semester tutoring students how to problem solve with programming.

Students came in from a variety of backgrounds, from majors to non-majors, freshmen to seniors, as well as people who had done some programming and total beginners. Learning to cater to each of their needs was a challenging task, but it's given me a better understanding of how to explain a concept or idea to different people.

Much of my work as a Course Assistant was helping the 1,277 students taking the course during Help Hours. Students hopped in a voice call with you, and you would clarify any confusion they had or try to lead them in the right direction, without giving away the answers. Aside from the 55 homework assignments, students also completed 3 machine problems, which guided them along making an app with a UI, a client, and a server with Android Studio. They had the choice of using Java or Kotlin for their assignments, so it was important to be flexible as a CA.

I learned a lot as a CA, and it gave me a new appreciation for explaining or documenting things well.
